\begin{itemize}
    \item \textbf{C++17：}提供高性能计算、内存精细控制与CUDA无缝互操作，避免Python运行时开销。

\end{itemize}

\section{锚点压缩场景表示模块}

\subsection{体素化锚点初始化}

锚点初始化的入口为`GaussianModel::initializeFromPoints`（实现于src/model/model\_core.cpp）。算法流程如下：

\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item \textbf{输入：}稠密点云$\mathcal{P}=\{(\mathbf{p}_j,\mathbf{c}_j)\}_{j=1}^{M}$（来自RGB-D初始帧或SFM前端）。
    \item 以 voxel\_size（典型值0.01 m）为步长对$\mathcal{P}$进行均匀体素化。
    \item 对每个非空体素，计算该体素内所有点的平均颜色$\bar{\mathbf{c}}$。
    \item \textbf{创建锚点：}
        \begin{itemize}
            \item $\mathbf{x}_i^a\leftarrow$ 体素中心坐标
            \item $\mathbf{f}_i^a\leftarrow$ 以平均颜色$\bar{\mathbf{c}}$初始化特征向量（通过简单线性映射至特征维度$D$）
            \item $\mathbf{o}_i^k\leftarrow$ 随机初始化$K=5$个子偏移
            \item $\mathbf{s}_i^a\leftarrow[0.01,0.01,0.01,\,s_v,s_v,s_v]$（前半控制offset范围，后半控制子高斯初始尺度$s_v$=体素尺寸）
        \end{itemize}
    \item 所有锚点参数存储为扁平GPU张量（torch::Tensor），索引统一的锚点id。
\end{enumerate}

\textbf{设计决策：}采用体素中心而非原始点位置作为锚点坐标，确保锚点在空间上的均匀覆盖，避免在点云稀疏区域出现空洞。

\subsection{增量锚点添加}

在SLAM运行过程中，新观测到的区域产生新点云时，调用`GaussianModel::addAnchors`（model\_core.cpp）增量添加锚点：

\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 将新点云体素化，筛选出未覆盖的体素。
    \item 以现有锚点位置为基准，执行半径过滤（radius-based duplicate filtering）：若新体素中心与任一现有锚点的距离小于 voxel\_size / 2，则跳过。
    \item 对通过筛选的新锚点，分配全局唯一递增ID，追加至各参数张量的末尾。
\end{enumerate}

\subsection{视角条件MLP解码器}

MLP解码器是锚点压缩的核心组件（实现于src/model/model\_mlp.cpp，类定义于include/scaffold\_chunks/anchor\_mlp.h）。三个独立MLP的输入$\mathbf{z}$完全相同：

\begin{equation}
\mathbf{z} = [\,\mathbf{f}_i^a\,|\,\mathbf{d}_{\text{view}}\,|\,\|\mathbf{x}_i^a - \mathbf{c}\|\,]
\end{equation}

其中$\mathbf{c}$为相机中心，$\mathbf{d}_{\text{view}}$为归一化视线方向，$\|\mathbf{x}_i^a - \mathbf{c}\|$为锚点至相机的欧氏距离。

\textbf{MLP结构：}

\begin{itemize}
    \item \textbf{mlp\_opacity\_}：$\mathbb{R}^{D+3+1}\to\mathbb{R}^{D}\to\mathbb{R}^{K}\to\operatorname{Tanh}$。输出$K$个tanh值，经sigmoid转换后作为$K$个子高斯的不透明度。
    \item \textbf{mlp\_cov\_}：$\mathbb{R}^{D+3+1}\to\mathbb{R}^{D}\to\mathbb{R}^{7K}$。输出$K$组$(\Delta s_x,\Delta s_y,\Delta s_z,\,q_w,q_x,q_y,q_z)$——3个尺度残差（与基础尺度相加）及4个旋转四元数分量（归一化后形成单位四元数）。
    \item \textbf{mlp\_color\_}：$\mathbb{R}^{D+3+1+A}\to\mathbb{R}^{D}\to\mathbb{R}^{3K}\to\operatorname{Sigmoid}$。若配置了外观嵌入维度$A>0$，则将每相机的外观嵌入向量拼接至输入。
\end{itemize}

\textbf{可选组件 —— mlp\_feature\_bank\_：}当 use\_feat\_bank 开启时，对锚点特征进行3分辨率注意力融合\cite{lu2024scaffoldgs}，提升视角适应能力。该功能默认关闭以控制参数总量。

\textbf{AnchorMLP::expandAnchorsToGaussians} 方法的处理流程：

\begin{enumerate}[label=\arabic*.,leftmargin=2em,itemsep=0pt,topsep=0pt]
    \item 输入：可见锚点子集 $[V, D]$ + 相机中心 $\mathbf{c}$
    \item 计算视线方向 $\mathbf{d}_{\text{view}} = \operatorname{normalize}(\mathbf{x}_a - \mathbf{c})$
    \item 计算逐锚点距离 $d_{\text{dist}} = \|\mathbf{x}_a - \mathbf{c}\|$
    \item 拼接输入 $\mathbf{z} = [\mathbf{f}_a \mid \mathbf{d}_{\text{view}} \mid d_{\text{dist}}] \rightarrow [V, D+4]$
    \item $\text{opacity\_raw} = \text{mlp\_opacity\_}(\mathbf{z}) \rightarrow [V, K]$
    \item $\text{cov\_raw} = \text{mlp\_cov\_}(\mathbf{z}) \rightarrow [V, K{\times}7]$
    \item $\text{color\_raw} = \text{mlp\_color\_}(\mathbf{z}) \rightarrow [V, K{\times}3]$
    \item 计算子高斯位置：$\mathbf{x}_i^k = \mathbf{x}_i^a + \mathbf{o}_i^k \cdot \mathbf{s}_i^a[:3]$
    \item 应用激活函数：opacity $\rightarrow$ sigmoid，scale $\rightarrow$ sigmoid，rot $\rightarrow$ normalize
    \item 生成 $\text{child\_mask}$：$\text{opacity} > 0$ 的子高斯
    \item 返回 $\text{ExpandedGaussians\{xyz, color, opacity, scaling, rotation, mask\}}$
\end{enumerate}

\subsection{层级化锚点生长与剪枝}

锚点生长与剪枝实现于src/model/model\_optimization.cpp。

\textbf{生长流程（`anchorGrowing`）：}

\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 统计每个锚点子偏移的累积梯度$\sum\|\nabla_{\mathbf{o}_i^k}\|$。
    \item 对梯度超过$\tau_{\text{grad}}=2\times10^{-4}$的锚点，以其高梯度子偏移位置为种子。
    \item 对种子进行 update\_depth=3 层级的体素细分：
        \begin{itemize}
            \item 第0层：体素尺寸$v_s\times\text{update\_init\_factor}=v_s\times100$
            \item 第$l$层：体素尺寸缩小$\text{update\_hierarchy\_factor}=4$倍
        \end{itemize}
    \item 在新体素中心创建新锚点（特征向量零初始化以保持初始稳定性）。
\end{enumerate}

\textbf{剪枝流程（`anchorPruning`）：}

对平均累积不透明度$\bar{\alpha}_i$低于$\tau_{\alpha}=0.005$的锚点执行剪枝：从所有参数张量中移除对应行，释放GPU内存。

\section{分块外存管理模块}

\subsection{空间分块与编码}

块的坐标系定义于include/scaffold\_chunks/chunk\_types.h。

\textbf{块坐标计算：}
\begin{equation}
c_i = \left\lfloor\frac{x_i}{S_{\text{chunk}}}\right\rfloor,\quad i\in\{x,y,z\}
\end{equation}

其中$S_{\text{chunk}}$为块边长（典型值20.0世界单位）。

\textbf{64-bit Chunk ID编码：}为支持高效哈希与序列化，将三元组$(c_x,c_y,c_z)$编码为单个64-bit整数：
\begin{equation}
\text{chunk\_id} = (c_x + 2^{20}) \cdot 2^{42} + (c_y + 2^{20}) \cdot 2^{21} + (c_z + 2^{20})
\end{equation}
每轴分配21 bits（含20-bit符号偏移），支持坐标范围$[-1,048,576,\;+1,048,575]$，对应场景规模$\sim$20,000 km（以$S_{\text{chunk}}=20$ m计），远超大场景SLAM需求。

\textbf{AABB包围盒：}每个块维护其轴对齐包围盒（Eigen::Vector3f min/max），用于视锥剔除的快速粗筛。

\subsection{LRU淘汰策略}

LRU管理器实现于src/model/model\_memory.cpp。

\textbf{核心数据结构：}
\begin{itemize}
    \item chunks\_loaded\_from\_disk\_：GPU暂存中已加载的块ID张量
    \item chunks\_on\_disk\_：已写至磁盘的块ID张量
    \item chunk\_access\_times\_：std::unordered\_map<int64\_t, time\_point>，记录每块的最近访问时刻
\end{itemize}

\textbf{淘汰流程（`evictExcessChunks`）：}

\begin{enumerate}[label=\arabic*.,leftmargin=2em,itemsep=0pt,topsep=0pt]
    \item $\text{protected\_ids} \leftarrow$ 当前帧可见的块ID
    \item $\text{excess} = \text{当前GPU锚点数} - \text{max\_anchors\_in\_memory}$
    \item if $\text{excess} \le 0$：return
    \item $\text{candidates} = \text{loaded\_chunks} \setminus \text{protected\_ids}$ \hfill // 非可见且已加载的块
    \item 按 $\text{chunk\_access\_times\_}$ 升序排序 $\text{candidates}$
    \item 选前 $N_{\text{evict}}$ 个块，其中 $N_{\text{evict}} = \text{excess} + 5\% \times \text{max\_anchors\_in\_memory}$
    \item 对每个选中块：
    \begin{enumerate}[label=\alph*.,leftmargin=3em,itemsep=0pt,topsep=0pt]
        \item $\text{saveAndEvictChunk(id)}$：序列化写入磁盘 + 从GPU张量中移除
        \item 更新 $\text{chunks\_loaded\_from\_disk\_} \;/\; \text{chunks\_on\_disk\_} \;/\; \text{chunk\_anchor\_counts\_}$
    \end{enumerate}
\end{enumerate}

\textbf{5\%滞后缓冲区设计：}避免在边界情况下频繁的"淘汰$\to$立即重新加载"震荡。当GPU锚点数降至$0.95\times M_{\max}$以下时，停止额外淘汰。

\textbf{内存压力自动监控（`checkMemoryPressure`）：}

训练循环的每次迭代调用此方法。当当前锚点数超过$M_{\max}$时，自动触发淘汰（无受保护块）。

\subsection{异步并行块I/O}

块读写使用异步并行I/O以最小化对训练循环的中断：

\begin{itemize}
    \item \textbf{`saveChunks`：}接收块ID列表，使用OpenMP并行启动写入任务，每个块在独立线程中将张量序列化为\schun 二进制流并写入磁盘。
    \item \textbf{`loadChunks`：}接收块ID列表，并行读取\schun 文件并重建为GPU张量。
\end{itemize}

\section{渲染与训练模块}

\subsection{两级视锥剔除}

视锥剔除实现于src/rendering/frustum\_culler.cpp，采用两级过滤：

\textbf{第一级 -- 块级AABB测试：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 从世界-视图-投影矩阵$\mathbf{WVP}$提取6个视锥平面$\{(\mathbf{n}_j,w_j)\}_{j=1}^6$（使用Gribb-Hartmann方法\cite{gribb2001fast}）。
    \item 对每个块的AABB进行保守的n-vertex测试：只要AABB存在至少一个顶点在所有平面内侧，则该块可见。
    \item 不可见块整体跳过——该块的锚点既不需加载也不参与MLP展开。
\end{enumerate}

\textbf{第二级 -- 锚点级球体测试：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 对可见块中的每个锚点，以其位置为球心、$R_i = \max_{k}\|\mathbf{o}_i^k\cdot\mathbf{s}_i^a[:3]\|$为半径构建包围球。
    \item 测试球心与6个视锥平面的符号距离：若任一距离$<-R_i$，则剔除该锚点。
\end{enumerate}

\textbf{内存管理集成：}剔除过程中，对判定为可见但尚未加载的块，自动触发`loadChunks`；对已加载的块，更新其访问时间戳。

最终输出一个布尔掩码`[N]`，标记所有可见锚点，供后续MLP展开与光栅化使用。

\subsection{双后端可微光栅化}

光栅化器实现于src/rendering/gaussian\_renderer.cpp。系统采用双后端设计，通过编译期宏HAVE\_CUDA\_RASTERIZER在两种渲染后端之间自动切换。

\textbf{渲染管线（`ScaffoldRenderer::render`）：}

\begin{enumerate}[label=\arabic*.,leftmargin=2em,itemsep=0pt,topsep=0pt]
    \item $\text{sliced\_anchors} = \text{model.getVisibleAnchors(visible\_mask)}$ \hfill // 切片可见锚点
    \item $\text{expanded} = \text{model.mlp().expandAnchorsToGaussians(...)}$ \qquad // MLP展开
    \item $\text{active} = \text{expanded.child\_mask}$ \hfill // 滤除不透明度 $\le 0$ 的子高斯
    \item 若 $\text{HAVE\_CUDA\_RASTERIZER}$ 已定义（INRIA CUDA光栅化）：
    \begin{quote}
        $\text{rgb, depth} = \text{RasterizeGaussians(}$\\
        $\quad\text{xyz[active], color[active], opacity[active],}$\\
        $\quad\text{scaling[active], rotation[active],}$\\
        $\quad\text{w2v, proj, FoVx, FoVy, H, W, bg)}$
    \end{quote}
    \item 否则（纯LibTorch回退）：
    \begin{quote}
        $\text{rgb, depth} = \text{renderGaussiansGPU(}$\\
        $\quad\text{xyz[active], color[active], opacity[active],}$\\
        $\quad\text{scaling[active], rotation[active],}$\\
        $\quad\text{w2v, proj, FoVx, FoVy, H, W, bg)}$
    \end{quote}
\end{enumerate}

\textbf{后端一 —— INRIA CUDA光栅化器：}
当CMake在third\_party/diff\_gaussian\_rasterization检测到子模块时，启用HAVE\_CUDA\_RASTERIZER宏，链接INRIA开源的tile-based可微高斯光栅化器。该后端实现了完整的3DGS渲染管线——将3D高斯经视图-投影矩阵变换后，基于tile排序进行精确的$\alpha$混合，渲染质量与原始3DGS论文一致。该子模块锁定在commit 59f5f77（2023年8月，代码冻结），由CMake直接编译其CUDA源文件（cuda\_rasterizer/*.cu）。

\textbf{后端二 —— 纯LibTorch GPU回退：}
当CUDA光栅化器不可用时，系统使用完全构建于LibTorch张量算子的GPU渲染器。该回退方案的核心流程为：

\begin{enumerate}
    \item \textbf{批量投影：}将所有3D点通过世界-视图-投影矩阵一次性变换至裁剪空间，再归一化至NDC空间与像素坐标。
    \item \textbf{视口过滤：}以GPU布尔掩码向量化过滤超出图像边界、相机后方及深度范围外的点，零CPU同步（仅.any().item<bool>()一次性判空）。
    \item \textbf{屏幕半径估算：}使用$r \approx f \cdot \bar{s} / z_w$近似计算每个高斯的投影屏幕半径（$f$为焦距，$\bar{s}$为平均尺度）。
    \item \textbf{深度排序与$\alpha$合成：}按像素平坦化索引排序高斯贡献后，使用torch::cumprod计算透射率乘积，通过scatter\_add逐像素累加加权颜色，实现前至后的$\alpha$混合。
\end{enumerate}

该回退方案所有操作均在GPU上执行，唯一的同步点为渲染前的空帧判出，在功能上完整支持开发与验证场景。但与INRIA原始tile排序实现相比，在排序精度、混合准确度和渲染效率方面存在一定差距，因此生产部署建议启用CUDA光栅化器后端。

\subsection{损失函数设计}

损失函数实现于src/training/losses.cpp，由四项加权组合：

\begin{equation}
\mathcal{L} = \mathcal{L}_1 + \lambda_{\text{SSIM}}\,\mathcal{L}_{\text{SSIM}}
              + \lambda_{\text{depth}}\,\mathcal{L}_{\text{depth}}
              + \lambda_{\text{iso}}\,\mathcal{L}_{\text{iso}}
\end{equation}

\textbf{L1颜色损失：}
\begin{equation}
\mathcal{L}_1 = \frac{1}{|\Omega|}\sum_{\mathbf{p}\in\Omega}\big|\hat{C}(\mathbf{p}) - C_{\text{gt}}(\mathbf{p})\big|
\end{equation}

\textbf{SSIM损失：}
\begin{equation}
\mathcal{L}_{\text{SSIM}} = 1 - \operatorname{SSIM}(\hat{C},\,C_{\text{gt}})
\end{equation}
使用基于高斯加窗的卷积实现结构相 似度（SSIM）度量\cite{wang2004ssim}。权重$\lambda_{\text{SSIM}}=0.2$。

\textbf{深度L1损失：}
\begin{equation}
\mathcal{L}_{\text{depth}} = \frac{1}{|\Omega_d|}\sum_{\mathbf{p}\in\Omega_d}\big|\hat{D}(\mathbf{p}) - D_{\text{gt}}(\mathbf{p})\big|
\end{equation}
权重$\lambda_{\text{depth}}=0.1$。仅在有效深度像素$\Omega_d$上计算。

\textbf{各向同性正则化：}约束子高斯尺度向各向同性方向收敛：
\begin{equation}
\mathcal{L}_{\text{iso}} = \frac{1}{M}\sum_{i=1}^{M} \|\mathbf{s}_i - \bar{s}_i\mathbf{1}\|_1
\end{equation}
其中$\bar{s}_i = \frac{1}{3}\sum_{j=1}^{3}s_{i,j}$。该正则化减少"针状"高斯伪影，权重$\lambda_{\text{iso}}=0.01$。

\subsection{训练调度器}

训练调度器`ScaffoldTrainer`实现于src/training/trainer.cpp，通过OpenCV FileStorage解析YAML配置文件（cfg/gaussian\_mapper/scaffold\_chunks.yaml）。

\textbf{初始化：}将配置参数映射至五组C++结构体（`AnchorModelConfig`、`ChunkConfig`、`OptimizationConfig`、`CameraConfig`、`KeyframeConfig`）。

\textbf{训练循环（`trainOneIteration`）：}

\begin{enumerate}[label=\arabic*.,leftmargin=2em,itemsep=0pt,topsep=0pt]
    \item $\text{kf} = \text{keyframe\_selector->getNextKeyframe()}$ \hfill // 基于损失加权采样
    \item 计算 WVP 矩阵 + 相机中心
    \item $\text{vis\_mask} = \text{model->cullVisibleAnchors(...)}$ \hfill // 两级剔除 + 块管理
    \item if $\text{vis\_mask.sum()} = 0$：skip iteration
    \item $\text{output} = \text{ScaffoldRenderer::render(...)}$ \hfill // MLP展开 + CUDA光栅化
    \item $\text{loss} = \text{computeAllLosses(output, kf->GT, weights)}$ \quad // L1 + SSIM + Depth + Iso
    \item $\text{loss.backward()}$ \hfill // 反向传播
    \item $\text{model->optimizerStep(visibility, N\_visible)}$ \hfill // Adam更新
    \item $\text{model->optimizerZeroGrad()}$ \hfill // 梯度清零
    \item if $\text{iter} \;\%\; \text{densify\_interval} = 0$：\hfill // 定期稠密化
    \begin{quote}
        $\text{model->adjustAnchors(...)}$\\
        $\text{model->trainingSetup(opt\_cfg)}$ \hfill // 重建优化器
    \end{quote}
\end{enumerate}

\textbf{关键帧采样器（`KeyframeSelection`）：}维护每帧的指数移动平均损失$\bar{\ell}_f$与被采样次数$u_f$，以概率：
\begin{equation}
P(f) \propto \frac{\bar{\ell}_f}{\max(u_f, 1)}
\end{equation}
进行离散采样。高损失、低使用率的帧有更高被选概率，促进训练均匀覆盖。

\subsection{深度估计模块}

深度估计模块实现于src/depth/depth\_estimator.cpp（接口定义于include/scaffold\_chunks/depth\_estimator.h），为系统提供统一的深度估计抽象层，支持两种深度源。

\textbf{单目深度估计（MonoDepthEstimator）：}
\begin{itemize}
    \item 设计为加载TensorRT推理引擎（如DepthAnything模型），接受RGB图像并输出稠密深度图。
    \item 通过initialize(engine\_path)加载TensorRT引擎并创建推理上下文。
    \item 完整的TensorRT推理管线已实现，生产环境中需配合预编译的DepthAnything引擎文件使用。
\end{itemize}

\textbf{双目深度估计（StereoDepthEstimator）：}
\begin{itemize}
    \item 基于OpenCV的cv::StereoSGBM实现，执行半全局块匹配（Semi-Global Block Matching）立体匹配。
    \item 深度计算：$d = b\cdot f_x / \delta$，其中$b$为基线长，$f_x$为焦距，$\delta$为视差值。
    \item 支持左右一致性检查（LR consistency check）以过滤遮挡区域的不可靠深度值，输出逐像素置信度掩码。
    \item 默认参数：基线0.12 m，焦距500像素，SGBM块大小5$\times$5，最小视差0，最大视差128。
\end{itemize}

\textbf{工厂函数：}createDepthEstimator(DepthSource, model\_path)根据枚举类型自动构建对应的深度估计器实例，便于系统在单目与双目模式间切换。

\subsection{SLAM前端——视觉跟踪}

视觉跟踪模块实现于src/slam/tracking.cpp（接口定义于include/scaffold\_chunks/tracking.h），提供帧间位姿估计的核心功能。

\textbf{ORB特征提取与匹配：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 对相邻帧提取ORB特征点（默认500个），计算BRIEF描述子。
    \item 使用FLANN匹配器进行描述子匹配，通过Lowe's ratio test（阈值0.7）滤除误匹配。
    \item 利用深度图将匹配的2D特征点反投影为3D点，构建3D-2D对应关系。
\end{enumerate}

\textbf{PnP位姿估计：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 使用EPnP + RANSAC从3D-2D对应关系中鲁棒估计相机位姿。
    \item RANSAC内点阈值设为2.0像素，最大迭代200次。
    \item 当深度信息不足时，回退至本质矩阵（5点法）+ 三角化恢复位姿。
\end{enumerate}

\textbf{运动模型：}维护恒定速度运动模型，当跟踪丢失时以模型预测值初始化新帧位姿。

\subsection{SLAM前端——回环闭合}

回环闭合模块实现于src/slam/loop\_closing.cpp（接口定义于include/scaffold\_chunks/loop\_closing.h），负责检测并修正长期累积漂移。

\textbf{场景识别（Place Recognition）：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 对每帧ORB描述子构建k-means词袋（BoVW）词汇表（默认10个聚类中心）。
    \item 对新关键帧，计算其BoVW直方图并与历史关键帧进行余弦相似度检索。
    \item 超过相似度阈值且间隔$\geq$50帧的候选帧进入几何验证阶段。
\end{enumerate}

\textbf{几何验证与位姿图优化：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 对候选回环帧，使用PnP + RANSAC验证3D-2D对应关系，计算相对位姿。
    \item 构建位姿图：节点为关键帧位姿，边为帧间约束（里程计边 + 回环边）。
    \item 在SE(3)流形上执行Gauss-Newton优化，最小化位姿图残差。
    \item 将优化后的位姿变换应用于受影响的子高斯位置，确保场景几何的一致性。
\end{enumerate}

\textbf{与Mapper的协作：}回环检测在BuiltinSLAMAdapter::detectAndEnqueueLoop中触发，通过MappingOperation(OprType::LoopClosingBA)进入Mapper的combineMappingOperations队列。处理回环时Mapper暂停帧接收（pause\_frame\_ingestion\_ = true），完成loop\_closure\_optimization\_iterations（默认1000次）次额外训练迭代后恢复。

\subsection{SLAM系统接口层}

SLAM系统接口层实现于include/scaffold\_chunks/slam\_system.h和src/slam/builtin\_adapter.cpp，是连接SLAM前端与高斯Mapper的抽象桥梁。

\textbf{SLAMSystemInterface：}定义统一的前端接口，包含trackMonocular、trackStereo、trackRGBD三种传感器模式的位姿估计接口，以及hasMappingOperation和getAndPopMappingOperation队列接口。该抽象使系统可以自由替换前端实现（从内置适配器到完整ORB-SLAM3集成）而无需改动Mapper。

\textbf{MappingOperation：}镜像ORB-SLAM3的Atlas操作类型，以OprType枚举区分LocalMappingBA（局部BA）、LoopClosingBA（回环BA）和ScaleRefinement（尺度精化）。每个操作携带关联的关键帧元组KFTuple（包含关键帧ID、相机ID、位姿$T_{cw}$、RGB图像、回环标志及辅助图像）。

\textbf{BuiltinSLAMAdapter：}是SLAMSystemInterface的自包含实现，内部持有TrackingModule和LoopClosingModule实例。trackMonocular/\\ trackStereo/ trackRGBD方法调用对应跟踪逻辑并返回位姿；detectAndEnqueueLoop方法定期（每10帧，ID > 50）执行回环检测并构造MappingOperation入队。

\subsection{SLAM Mapper管线（DiskChunGS对齐）}

Mapper管线实现于src/mapper/mapper\_core.cpp（接口定义于include/scaffold\_chunks/gaussian\_mapper.h），采用DiskChunGS对齐的两阶段设计，作为系统的顶层编排器。

\textbf{架构设计：}Mapper提供两种构造模式——SLAM模式（接收SLAMSystemInterface共享指针）与外部模式（接收外部位姿输入），使系统可灵活切换自主SLAM或离线全局优化模式。

\textbf{两阶段运行循环（run()）：}

\textbf{Phase 1 —— 初始建图：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 循环处理传入帧数据：若处于SLAM模式，通过slam\_system\_->trackStereo()等接口获取位姿；若为外部模式，直接使用传入位姿。
    \item 将每帧封装为GaussianKeyframe，累积至GaussianScene容器。
    \item 当满足初始建图条件（累积$\geq$min\_num\_initial\_map\_kfs帧）时，从SLAM系统拉取并处理待处理的MappingOperation。
    \item 为所有关键帧批量生成高斯种子（sampleGaussians），调用model\_->trainingSetup初始化优化器。
    \item 执行一次训练迭代后进入Phase 2。
\end{enumerate}

\textbf{Phase 2 —— 增量映射：}
\begin{enumerate}[label=(\arabic*),leftmargin=2em]
    \item 持续接收新帧，由SLAM系统判定是否生成新关键帧及MappingOperation。
    \item 定期调用combineMappingOperations()：将所有队列中的操作按类型分组（LocalBA、LoopBA、ScaleRefinement）。
    \item LocalBA批量处理：从操作关联的关键帧中采样高斯并进行局部窗口训练。
    \item LoopBA处理：暂停帧接收（100ms缓冲），应用回环变换至受影响的子高斯，执行额外loop\_closure\_optimization\_iterations（默认1000）次训练。
    \item 每次迭代执行trainForOneIteration()：选择关键帧 $\to$ 视锥剔除 $\to$ MLP展开 $\to$ 渲染 $\to$ 损失计算 $\to$ 反向传播 $\to$ Adam更新 $\to$ 定期稠密化。
\end{enumerate}

\textbf{回环闭合内存管理：}在回环闭合期间，loop\_closure\_memory\_multiplier\_（默认8.0$\times$）将GPU锚点上限临时扩大，以容纳回环区域可能加载的额外块，避免在关键优化阶段触发频繁淘汰。

\subsection{实时可视化模块}

实时可视化模块实现于src/viewer/gaussian\_viewer.cpp（接口定义于include/scaffold\_chunks/gaussian\_viewer.h），提供独立线程运行的交互式3D场景查看器。

\textbf{渲染架构：}
\begin{itemize}
    \item 基于GLFW窗口 + OpenGL 3.3核心上下文，以独立线程运行，与Mapper线程解耦。
    \item 双缓冲ViewerFrameData结构：Mapper线程写入当前帧数据快照（高斯点云、相机位姿、关键帧列表），Viewer线程以互斥锁保护读取。
    \item 绘制三类元素：相机视锥（绿色线段表示图像平面四角+相机中心连线）、关键帧轨迹（橙色折线连接所有KF位置）、高斯点云（按颜色的3D包围球/点）。
\end{itemize}

\textbf{交互控制：}
\begin{itemize}
    \item \textbf{轨道相机：}鼠标左键拖动旋转（方位角+仰角）、中键拖动平移、滚轮缩放。
    \item \textbf{ImGui控制面板：}实时显示FPS、锚点数量、训练迭代数；提供渲染开关（视锥/点云/网格）、点大小滑块、背景色设置。
    \item 默认窗口分辨率1920$\times$1080，背景深灰（RGB 0.15, 0.15, 0.15）。
\end{itemize}

\textbf{与Mapper的数据交互：}Mapper通过submitFrameData()接口推送每帧快照；Viewer仅读取快照副本，不直接访问GaussianModel或GaussianScene，确保线程安全。

\section{关键工程实现细节}

\subsection{YAML配置系统}

配置文件使用OpenCV YAML格式（cv::FileStorage），定义了五组结构化配置节点（Anchor、Chunk、Optimization、Camera、Keyframe），总计40+可调参数。C++ struct与YAML节点间通过键名字符串映射实现自动解析（trainer.cpp中的`fromYAML`函数）。

\subsection{线程安全设计}

关键帧容器（`GaussianScene`）使用std::mutex保护内部std::map，提供线程安全的`addKeyframe`/`getKeyframe`/`getAllKeyframes`接口，确保训练线程与数据加载线程的并发安全。

\subsection{张量序列化}

`TensorHeader`提供跨平台的张量元数据编码：维度数、各维度大小、数据类型（c10::ScalarType的整数编码）及数据字节数。序列化时直接写入tensor.data\_ptr()指向的原始内存块，反序列化时通过torch::from\_blob重建张量，避免逐元素拷贝。

\section{本章小结}

本章详细阐述了\ScaffoldChunGS 系统的设计与实现。系统以十大模块——锚点压缩表示、分块外存管理、双后端渲染、多目标训练、深度估计（SGBM+TensorRT）、视觉跟踪（ORB+PnP）、回环闭合（BoVW+SE(3)）、SLAM系统接口（SLAMSystemInterface+MappingOperation）、两阶段DiskChunGS对齐Mapper管线与实时可视化——为核心，在C++17/CUDA/LibTorch技术栈上构建了完整的3D高斯SLAM管线。核心设计决策包括：锚点粒度块管理使压缩与扩展机制对齐、双后端渲染策略（INRIA CUDA光栅化器 + 纯LibTorch回退）兼顾性能与可移植性、两级视锥剔除减少无效计算、LRU+5\%滞后策略平衡性能与稳定性、自包含\schun 格式支持训练持久化、DiskChunGS对齐的两阶段Mapper（Phase 1初始建图+Phase 2增量映射）及其MappingOperation分发机制。第四章将通过实验验证系统的有效性。
